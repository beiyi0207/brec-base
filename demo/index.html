Grid

Unlike other grid systems, we don't give you any classes to put in your markup. Instead, you're required to @extend our placeholder selectors like so:

.hero {
  @extend %all;
  @extend %columns;
}

.content {
  @extend %nine;
  float: left;
}

  .info-column {
    @extend %four;
    @extend %columns;
  }

.sidebar {
  @extend %three;
  @extend %columns;
}

So, while this doesn't force you to use semantic class names, it at least forces you to think about how you're naming your classes. With the above Sass, your markup will look nice and clean as well:

<section class="hero"></section>
<section class="content">
  <div class="info-column"></div>
  <div class="info-column"></div>
  <div class="info-column"></div>
</section>
<aside class="sidebar"></aside>

Icons

We are only including icons that are needed by our modules, a subset of Font Awesome by Dave Gandy - http://fontawesome.io. Many CSS frameworks include giant icon libraries, when you really only need a few. So instead of serving the end user bigger than necessary icon font and CSS files, try curating them with one of several free tools online. We use Icomoon. Feel free to import /fonts/brecons.json into Icomoon to use our small selection of icons as a base for your own icon library.

To use, include /base/_icons.scss and include your font path (relative to your generated CSS file):

@include icons('../fonts');

Will generate:

@font-face {
  font-family: 'brec';
  src:url('../fonts/brec.eot');
  src:url('../fonts/brec.eot#iefix') format('embedded-opentype'),
    url('../fonts/brec.woff') format('woff'),
    url('../fonts/brec.ttf') format('truetype'),
    url('../fonts/brec.svg#brec') format('svg');
  font-weight: normal;
  font-style: normal;
}

Use icons in your markup like so:

<span class="icon-square-checked"></span>

Notice the use of <span></span> as opposed to Bootstrap's <i></i> convention. We highly recommend this because WCAG 2.0 does not allow the use of the <i> tag.

Radio Buttons as Buttons

Given a small selection of options, it may be a good idea to use radio buttons styled as a the actual buttons they're named after. Just put the radio buttons inside a <label> with the .button class and wrap them all in a .button-group.

<fieldset>
  <fieldset class="first">
    <legend>What floats in water?</legend>
    <input type="radio" name="what-floats" id="what-floats-bread" value="bread"><label for="what-floats-bread" class="button">Bread</label>
    <input type="radio" name="what-floats" id="what-floats-rocks" value="very small rocks"><label for="what-floats-rocks" class="button">Very Small Rocks</label>
    <input type="radio" name="what-floats" id="what-floats-churches" value="churches"><label for="what-floats-churches" class="button">Churches</label>
    <input type="radio" name="what-floats" id="what-floats-duck" value="duck"><label for="what-floats-duck" class="button">A Duck</label>
  </fieldset>
</fieldset>

Navigation

Our nav.scss includes one CSS rule, that for a.skip-link. This is used for an accessible "Skip Navigation" or "Skip to Content" link before your main navigation. See http://webaim.org/techniques/skipnav/ for an explaination on the why and how of this.
